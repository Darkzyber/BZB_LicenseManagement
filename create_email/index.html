เปลี่ยน flow เดิมจาก trigger "When SP item created" → When HTTP request received เพื่อให้ caller flow 3 กรณีเรียกใช้ module เดียวกันได้ โดย logic แกนกลางไม่เปลี่ยน

🛡️ SP Admin Request Flow
trigger: approval approved
caseType: adminSP

CHILD FLOW (MODULE)
HRIS Create Account
⚡ HTTP trigger
👤 Get Employee

Generate UPN

แยก format ตาม caseType — ใช้ Switch action

🧑‍💼 newHire
first_name.XX@buzzebees.com
substring lastname 2 ตัว
loop เพิ่มทีละตัวถ้าซ้ำ
📦 centralMail / adminSP
first_name.XXXX@buzzebees.com
substring lastname 4 ตัว
loop เพิ่มทีละตัวถ้าซ้ำ
expressionSet variable: mail
// newHire — เริ่มที่ 2 ตัว
concat(
  toLower(replace(FirstName, ' ', '_')),
  '.',
  substring(toLower(LastName), 0, add(variables('Counter'), 1)),
  '@buzzebees.com'
)
// Counter=1 → 2 ตัว, Counter=2 → 3 ตัว, ...

// centralMail / adminSP — เริ่มที่ 4 ตัว
concat(
  toLower(replace(FirstName, ' ', '_')),
  '.',
  substring(toLower(LastName), 0, add(variables('Counter'), 3)),
  '@buzzebees.com'
)
// Counter=1 → 4 ตัว, Counter=2 → 5 ตัว, ...
Edge case: ถ้า lastname สั้นกว่า 2 ตัว (หรือ 4 ตัวสำหรับ central) ให้ fallback ใช้ lastname เต็ม + running number ต่อท้าย เช่น first_name.lee1@buzzebees.com

UPN Algorithm ตัวอย่าง

เปรียบเทียบ algorithm เดิม vs ใหม่

ของเดิม: substring(lastName, Counter-1, 2) → ตัดจาก index ที่เลื่อนไป ตัวเดิมแค่ 2 ตัวเสมอ → ชนกันได้ง่าย และถ้า Counter เกิน = index out of range
ใหม่: substring(lastName, 0, Counter+1) → เริ่มจากหน้าเสมอ แต่ยาวขึ้นเรื่อยๆ → ยิ่ง loop ยิ่ง unique
ตัวอย่าง — lastname: sombatrak
newHire
john.so
john.som
john.somb ✓
central
john.somb
john.somba ✓
taken = UPN นั้นมีอยู่ใน SP Entra Log แล้ว | ✓ = ใช้ได้
Fallback: ถ้า Counter > length(lastName) ทั้งหมด → สร้าง UPN แบบ fname.lname{timestamp4}@buzzebees.com แล้ว terminate พร้อม alert ให้ HRIS ดูด้วยตัวเอง

ส่วนเดิมใหม่Status
