⚡ Power Automate — 4 Flows
4 Flows ที่ชัดเจน แต่ละ Flow มีหน้าที่เดียว — Flow 1–2: Request/Purchase, Flow 3: Expiry Action, Flow 4: License Movement (Priority 0)
🎯
Priority 0 — Core Requirement: Flow 3 (Expiry Action) และ Flow 4 (License Movement) คือ Requirement หลักที่ได้รับมอบหมาย ไม่ใช่แค่ nice-to-have เรื่อง Resign/Transfer license ต้องไม่หลุดลอย และ Expiry ต้องมี Action จริงๆ ไม่ใช่แค่แจ้งเตือน
✅
หลักการ: 1 Flow = 1 หน้าที่ ไม่ combine notification หลายอย่างในที่เดียว และไม่แยก Flow สำหรับทุก notification เล็กๆ ใช้ branch ใน Flow เดิมแทน
⚡ Flow 1 — Request & Approval ปรับ — เพิ่ม PaymentMethod routing
📦
Package License Pool: Flow 1 ต้องเช็ค FreeSeats ก่อนเสมอ (Step 0) — ถ้ายังมี seat ว่างในระบบให้ Assign ทันที ไม่ต้องผ่าน Approval หรือ Purchase เลย ประหยัดทั้งเวลาและ process
StepActionหมายเหตุ
TriggerPowerApps V2 triggerรับข้อมูลจาก Request Form (SoftwareRef, RequestedBy, LicenseQty)
0 🆕[Pool Check] Get item → SoftwareCatalog (SoftwareRef จาก Form)
→ Condition: FreeSeats > 0 ?
YES (Pool Available): Update Licenses (AssignedTo = Employee, Status = "Active") → Create LicenseHistory log (Action="Pool Assign") → Send Email Employee "License พร้อมใช้งานแล้ว" → Terminate (ไม่ต้องผ่าน Purchase)
NO (Pool Empty): ดำเนินการต่อ Step 1 ปกติ		⚡ ถ้า Pool ว่าง → ข้าม Phase 2 ทั้งหมด บันทึก log ว่า "Assigned from Pool"
1Create item → ApprovalRequests (รวมถึง PaymentMethod, PurchaseStatus="Pending")ทำเมื่อ Pool หมดเท่านั้น
2Start and wait for an approval → ส่งให้ Manager"Approve/Reject – First to respond"
3a ✅If Approved: Update item → ApprovalStatus="Approved", PurchaseStatus="In Process"—
3b ✅Switch(PaymentMethod):
· "PR/PO" → Send Email แผนกจัดซื้อ + link Power Apps Admin
· "Corporate Card" → Send Email แผนกการเงิน + link Power Apps Admin		Email แนบรายละเอียด Request + link กรอก write-back
4a ❌If Rejected: Update item → ApprovalStatus="Rejected", PurchaseStatus="Cancelled"—
4b ❌Send Email to Employee (แจ้งผลปฏิเสธ + comments)ดึง Comments จาก Approval response
⚡ Flow 2 — Purchase Complete Trigger ใหม่ (แทน Flow 4+5 เดิม)
🔑
นี่คือ Write-Back Listener — คอยดูว่าเมื่อไหร่ PurchaseStatus เปลี่ยนเป็น "Completed" แล้วเริ่ม Phase 3 ทันที
StepActionหมายเหตุ
TriggerWhen an item is modified → ApprovalRequests—
1Condition: PurchaseStatus eq 'Completed' → ถ้าไม่ใช่ Terminateกัน Flow ทำงานโดยไม่จำเป็น

3Send Email to HR/IT Admin: "คำขอ [Title] จัดซื้อเสร็จแล้ว กรุณา assign License และกรอก LicenseAccountEmail"
แนบ: Reference = if(empty(PONumber), CCRefID, PONumber)		Link ไปหน้า Admin Power Apps
3b 🆕[HRIS Account Request] Send Email to HRIS team: "กรุณาสร้าง License Account สำหรับ [Employee Name] — ซอฟต์แวร์ [Title]"
Pattern: firstname.surname4chars@buzzebees.com
เช่น พีรพล จุลสุวรรณ → peerapol.chun@buzzebees.com
หลังสร้างแล้ว HRIS กรอก LicenseAccountEmail ใน Licenses list ผ่าน Power Apps		👤 HRIS Account แยกจาก AD Account พนักงาน — กรณีลาออกไม่ต้อง Cancel ที่ Vendor แค่ Reset Password
4Create item → LicenseHistory (Action="Purchase Completed", Reference=PONumber หรือ CCRefID)audit trail
⚡ Flow 3 — Expiry Action Flow อัปเกรด — จาก "แค่แจ้งเตือน" เป็น "บังคับให้ตัดสินใจ"
🎯
Priority 0: Expired ≠ แค่ส่ง Email แล้วจบ — ต้องมี Task ที่ IT Admin ต้อง Action จริงๆ (ไปยกเลิก/ต่ออายุที่ Vendor Portal) และต้องมีคนรับผิดชอบชัดเจน ไม่ใช่แค่ "ส่งเมลแจ้ง"
StepActionหมายเหตุ
TriggerRecurrence — ทุกวัน 08:00 น.—
1Get items → Licenses (filter: Status eq 'Active' and ExpiryAction ne 'Done')กัน License ที่ตัดสินใจแล้ว
2Apply to each: คำนวณ DaysLeft
div(sub(ticks(ExpiryDate),ticks(utcNow())),864000000000)		—
3aDaysLeft = 30: Update ExpiryActionStatus="Pending Decision"
→ Send Email: License Owner + Manager + IT Admin
→ เนื้อหา: "License [Name] หมดอายุใน 30 วัน กรุณาเลือก [ต่ออายุ / ยกเลิก / โอนย้าย] ผ่าน Power Apps ภายใน 7 วัน"		Email มี link ตรงไปหน้า Admin ของ Power Apps
3bDaysLeft = 7 และ ExpiryActionStatus = "Pending Decision":
→ Escalate ส่ง Email ซ้ำ IT Admin + Manager: "ยังไม่มีการตัดสินใจ License จะหมดใน 7 วัน"		กันกรณีที่ไม่มีใค Action เลย
3cDaysLeft = 1 และ ExpiryActionStatus ≠ "Done":
→ Update ExpiryActionStatus="Action Required"
→ Send Email IT Admin: "License หมดพรุ่งนี้ — ดำเนินการที่ Vendor Portal ทันที"		IT Admin คือคนที่ต้องลงมือจริงที่ Vendor Portal
4DaysLeft ≤ 0: Update Status="Expired"
→ Create LicenseHistory log (Action="Expired")
→ Send Email IT Admin: "License [Name] หมดอายุแล้ว — ยืนยันการดำเนินการที่ Vendor Portal และ mark Done ใน Power Apps"		ถ้ายังไม่ Cancel ที่ Vendor = อาจถูก Auto-renew และเสียเงิน
👤
ใครมีอำนาจเข้า Vendor Portal: IT Admin เท่านั้น — เพราะ License ผูกกับ Billing Email ของบริษัท (ไม่ใช่ Email พนักงาน) Manager มีอำนาจ ตัดสินใจ (Renew/Cancel) แต่ IT Admin คือคนที่ ลงมือ ที่ Vendor Portal จริงๆ
⚡ Flow 4 — License Movement ใหม่ — Priority 0
🎯
Priority 0: เมื่อพนักงาน Resign หรือ Transfer แผนก License ต้องไม่หลุดลอย — ต้องมีคนรับไป (Transfer) หรือคืน Pool หรือยกเลิก ก่อนที่ IT จะปิด Account พนักงาน
StepActionหมายเหตุ
TriggerPowerApps V2 trigger
(IT Admin กด "ดำเนินการ Movement" ใน Admin Screen)		รับ: LicenseID, MovementAction, TransferToEmail, Reason
1Get item → Licenses (LicenseID ที่ส่งมา)ดึงข้อมูล License ปัจจุบัน
2aIf MovementAction = "Transfer":
Update Licenses: AssignedTo=TransferToEmail, MovementStatus="Transferred", MovementReason=Reason, TransferredTo=TransferToEmail
→ Send Email ผู้รับ License ใหม่: "คุณได้รับ License [Name] โอนมาจาก [เดิม]"
→ Send Email Manager: "License ของ [พนักงานเดิม] โอนให้ [ใหม่] แล้ว"		License ยังใช้งานต่อ ไม่ขาดตอน
2bIf MovementAction = "Return to Pool":
Update Licenses: AssignedTo=blank, MovementStatus="Returned to Pool", MovementReason=Reason
→ Send Email IT Admin + Manager: "License [Name] คืน Pool แล้ว รอ Assign ใหม่"		License ยังอยู่ในระบบ ไม่ถูกยกเลิก
2cIf MovementAction = "Cancel":
Update Licenses: Status="Cancelled", MovementStatus="Cancelled", MovementReason=Reason, ExpiryAction="Cancel"
→ Create LicenseHistory log (Action="Cancelled - "+Reason)
→ Send Email IT Admin: "License [Name] ถูก Cancel — ดำเนินการ Cancel ที่ Vendor Portal ด้วย"		IT Admin ต้องไป Cancel ที่ Vendor Portal จริงๆ ด้วย
3Create LicenseHistory log ทุก path (Action=MovementAction, ChangedBy=IT Admin, Note=Reason)Audit trail ครบ
⚠️
ข้อควรระวัง: Flow นี้ต้อง run ก่อนที่ IT Admin จะปิด Account พนักงาน (Disable AD/Entra account) ถ้าปิด Account ก่อน License จะหลุดลอยทันที เพราะ AssignedTo อ้างถึง account ที่ disabled แล้ว

🔀 กรณีที่ 1: License Movement

TRIGGER

3 ACTIONS ที่เลือกได้

ผล: AssignedTo เปลี่ยน, ใช้งานต่อได้ทันที
